void mdtApplication::changeLanguage ( const QLocale &  locale = QLocale()) [slot]

Change the application language.

This will remove installed translators, and install language speficifed translators. Whenn translation files could not be loaded, the application should fall back to source code's language. If no parameters are given, system's language will be used. By default, the .qm translation files are searched in system's data directory. It's possible to add other search paths using otherQmDirectories parameter.

Precondition:
The error system must be initalized before using this method (this is done by init() ).

bool mdtApplication::init ( bool  allowMultipleInstances = false,
int  dialogErrorLevelsMask = 0 
)

Some initializations.

Several things are done here:

  • Search the system data directory
  • Init the error system
  • Check if another instance is running (if allowMultipleInstances Instance is false)
  • Load translation files and set language according system settings.
    Parameters:
    allowMultipleInstancesIf true, multiple instance of application is allowed (but NOT multiple instance of mdtApplication in program !)
    dialogErrorLevelsMaskSet the mask of errors that are displayed as dialogs in GUI. By default, no error (geven by mdtErrorOut) are displayed in GUI.

void mdtApplication::installTranslator ( QTranslator *  translator)

Adds the translator to the list of translators to be used for translations.

Note: this method overloads QCoreApplication::installTranslator() . The difference is that it adds new translator to a list, and delete it all in destructor. Internally, it calls QApplication::installTranslator(). To keep trace of installed translators, you should allways use this method, and never mix calls with, for example, qApp->installTranslator() or QCoreApplication::installTranslator()

Precondition:
translator must point to a valid object.
