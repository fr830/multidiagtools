Send a query to device.

Wait until it's possible to write to device, send the query and wait until a response is available or timeout.

Note that the wait will not break the GUI's event loop.

This method will not update internal I/O states. It's recommended to use mdtDevice functions if available.

Parameters:
queryQuery to send.
writeTimeoutWrite timeout [ms]
readTimeoutResponse timeout [ms]
Returns:
Result as string (empty string on error) Note that mdtFrameCodecScpi can be helpful to decode returned result.
