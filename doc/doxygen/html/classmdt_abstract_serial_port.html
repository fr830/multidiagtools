Interface class for serial port

This is a abstract class used to build system specific implementation.
The subclass should have mdtSerialPort as name, so it will be compatible with several classes (f.ex. mdtSerialPortManager).
To begin a implementation, please take a look at mdtAbstractPort at first.

See also:
mdtAbstractPort

virtual void mdtAbstractSerialPort::buildAvailableBaudRates ( ) [protected, pure virtual]

Build the list of available baud rates.

The subclass must search about available baud rates for current port, and should check if it can be really be used. For.ex. , on POSIX, termios defines some baud rates, but all are not necessarily supported by port. To be shure, a solution is to try each baudrate (with cfsetispeed()/cfsetospeed() on POSIX) before storing it in container. Note that the precondition must be checked with Q_ASSERT(isOpen())

Precondition:
Port must be open before this call.

virtual void mdtAbstractSerialPort::mapUartType ( ) [protected, pure virtual]

Map the system defined UART type to internal one.

In this class, the UART type is defined in sp_uart_type_t enum. The subclass must map the system's internal type to sp_uart_type_t. On error (f.ex. by system calls), the method should use the mdtError system, and set the UART type to UT_UNKNOW. Note that the precondition must be checked with Q_ASSERT(isOpen())

Precondition:
Port must be open before this call.

virtual error_t mdtAbstractSerialPort::waitEventCtl ( ) [pure virtual]

Wait until a control (modem line) signal state changes.

This method is called from mdtSerialPortCtlThread , and should not be used directly.
Mutex must be locked before calling this method with lockMutex(). The mutex is locked when method returns.

Subclass notes:
This method must be re-implemented in plateform specific subclass.
Notes about mutex handling:

  • Mutex must be released during wait, and relocked befor return.
Returns:
On success, NoError is returned. If waiting is canceled, WaitingCanceled is returned and the thread knows that it must end (case of stopping thread). On unhandled error, UnknownError is returned. The thread also stop working, and emit a error signal (that can be handled in mdtPortManager, or in another place in application).

mdtPortThread
