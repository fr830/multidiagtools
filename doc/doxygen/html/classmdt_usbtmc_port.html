Detailed Description

USBTMC device file calss.

With USBTMC, when we try to read, and no data are available, both read and write threads are blocked.
To start the read, call the readOneFrame() method.
To start the write, call the writeOneFrame() method.

void mdtUsbtmcPort::readOneFrame ( ) [virtual, slot]

Tell the thread that a frame can be read.

In mdtUsbtmcPortManager, this slot is connected to the write thread, and called each time a frame was written.
If the waitingFrame flag is true, the read thread will be notified that data must be read.
